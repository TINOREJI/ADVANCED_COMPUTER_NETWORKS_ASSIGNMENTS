# 💻 ⚙️ Configuration Commands


## 🔧 Router R1 (AS 100)

```bash
enable
configure terminal

interface g0/0
ip address 192.168.1.1 255.255.255.0
no shutdown
exit

interface g0/1
ip address 10.0.0.1 255.255.255.252
no shutdown
exit

router bgp 100
neighbor 10.0.0.2 remote-as 200
network 192.168.1.0 mask 255.255.255.0
```


## 🔧 Router R2 (AS 200)

```bash
enable
configure terminal

interface g0/0
ip address 10.0.0.2 255.255.255.252
no shutdown
exit

interface g0/1
ip address 10.0.0.5 255.255.255.252
no shutdown
exit

router bgp 200
neighbor 10.0.0.1 remote-as 100
neighbor 10.0.0.6 remote-as 300
network 10.0.0.0 mask 255.255.255.252
```


## 🔧 Router R3 (AS 300)

```bash
enable
configure terminal

interface g0/0
ip address 10.0.0.6 255.255.255.252
no shutdown
exit

interface g0/1
ip address 192.168.3.1 255.255.255.0
no shutdown
exit

router bgp 300
neighbor 10.0.0.5 remote-as 200
network 192.168.3.0 mask 255.255.255.0
```

## 📊 Key Concepts

* **BGP (Border Gateway Protocol)** → Used between different networks (AS)
* **AS (Autonomous System)** → Independent network
* **eBGP** → Between different AS
* **AS_PATH** → Prevents loops
* **NEXT_HOP** → Next router in path

## 🔍 Observations

* Routes are exchanged between routers dynamically
* Each router learns about remote networks
* Communication is established between different AS

## 💡 What I Learned

* BGP is used for Internet-level routing
* It is more complex than RIP and OSPF
* Routing decisions are based on path attributes
* Each router acts as an independent system
