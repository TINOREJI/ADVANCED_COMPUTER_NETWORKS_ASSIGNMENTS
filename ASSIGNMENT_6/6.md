## 📘 About This Assignment

This project demonstrates how to configure **dynamic routing using RIP Version 2** in **Cisco Packet Tracer**.

Instead of manually defining routes (like static routing), RIP automatically shares routing information between routers.


## 🎯 Objective

* Configure IP addressing on routers and PCs
* Enable RIP Version 2
* Advertise networks dynamically
* Disable auto-summary
* Verify connectivity across networks


## 🧠 Network Overview

The network consists of:

* 3 Routers (R1, R2, R3)
* 2 LAN networks
* Serial connections between routers

RIP is used to allow communication between all networks.

## 🛠️ IP Addressing

| Device | Network       |
| ------ | ------------- |
| R1–R2  | 100.1.1.0/24  |
| R1–R3  | 20.1.1.0/24   |
| LAN 1  | 172.16.2.0/16 |
| LAN 2  | 10.2.2.0/24   |


## ⚙️ Configuration

### 🔹 Enable RIP on Routers

```bash id="ripreadme"
router rip
version 2
no auto-summary
```

### 🔹 Advertised Networks

* R1 → 100.1.1.0, 20.1.1.0
* R2 → 100.1.1.0, 172.16.0.0
* R3 → 20.1.1.0, 10.2.2.0


## 💻 PC Configuration

**PC1**

```text id="pcr1"
IP: 172.16.2.10
Gateway: 172.16.2.1
```

**PC2**

```text id="pcr2"
IP: 10.2.2.20
```

Dynamic routing using RIP Version 2 was successfully configured, allowing communication between all networks.


## 💡 What I Learned

* RIP automatically updates routing tables
* Version 2 supports subnetting (classless routing)
* Auto-summary must be disabled for correct routing
* Dynamic routing reduces manual work


## 📌 Final Thoughts

This experiment helped understand how routers dynamically learn routes instead of relying on manual configuration. It’s a key step towards understanding advanced routing protocols like OSPF and EIGRP.
