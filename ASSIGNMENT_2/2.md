## 📘 About This Assignment

This assignment focuses on analyzing different network protocols using **Wireshark**.

Instead of just studying theory, this experiment allows us to observe how protocols like ICMP, TCP, UDP, DNS, HTTP, and ARP work in real-time.

---

## 🎯 Objectives

* Capture and analyze **ICMP packets (ping)**
* Understand **TCP three-way handshake**
* Analyze **UDP communication**
* Capture **DNS queries and responses**
* Observe **HTTP request/response**
* Analyze **ARP request and reply**
* Identify **MAC and IP addresses**

---

## 🛠️ Requirements

* Wireshark installed
* Internet connection
* Command Prompt / Terminal

## 🔍 Protocol Analysis

---

### 🔹 ICMP (Ping)

* Echo Request → Sent
* Echo Reply → Received

💡 Used for connectivity testing

---

### 🔹 TCP (Three-Way Handshake)

```text
SYN → SYN-ACK → ACK
```

💡 Ensures reliable communication

---

### 🔹 UDP

* No connection setup
* No acknowledgment

💡 Fast but unreliable

---

### 🔹 DNS

* Query → Domain name
* Response → IP address

💡 Resolves domain to IP

---

### 🔹 HTTP

* GET request
* Server response (200 OK)

💡 Used for web communication

---

### 🔹 ARP

* Request → “Who has IP?”
* Reply → “Here is MAC”

💡 Maps IP → MAC address
