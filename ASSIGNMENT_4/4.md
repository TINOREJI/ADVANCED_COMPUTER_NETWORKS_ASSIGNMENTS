## 📘 About This Assignment

This assignment focuses on configuring a **DHCP (Dynamic Host Configuration Protocol)** server on a router using **Cisco Packet Tracer**.

Instead of manually assigning IP addresses, the router dynamically provides IP configuration to client devices.

---

## 🎯 Objectives

* Configure a DHCP pool named **xyz**
* Assign IP addresses dynamically to clients
* Configure DNS and lease time
* Disable BOOTP service
* Verify automatic IP assignment


## 🛠️ Network Setup

* 1 Router (DHCP Server)
* 1 Switch
* 2 Client PCs

All devices are connected in a simple LAN.

## ⚙️ Configuration Steps

### 1️⃣ Configure Router Interface

Assign IP to router:

```bash
interface Gig0/0/0
ip address 192.168.10.1 255.255.255.0
no shutdown
```
<img width="503" height="95" alt="image" src="https://github.com/user-attachments/assets/31e6185f-98d8-4680-a7a8-67e39ec27c06" />

### 2️⃣ Create DHCP Pool

```bash
ip dhcp pool xyz
network 192.168.10.0 255.255.255.0
default-router 192.168.10.1
dns-server 1.2.3.4
lease 0 3(Not Supported)
```
<img width="453" height="87" alt="image" src="https://github.com/user-attachments/assets/708516c1-a94d-4008-93f9-4ae5c984823c" />

### 3️⃣ Disable BOOTP

```bash
no ip bootp server
```

### 4️⃣ Configure Clients

Set IP configuration to:

```text
DHCP (Automatic)
```
<img width="524" height="319" alt="image" src="https://github.com/user-attachments/assets/b2e7ed8d-987f-40c7-ba01-6ccc7533a12e" />

## 🔍 Observations

* Clients received IP addresses automatically
* IP range assigned: `192.168.10.x`
* DNS server assigned: `1.2.3.4`

## ✅ Result

The DHCP server was successfully configured on the router, and client devices were able to obtain IP addresses automatically along with DNS and gateway settings.


## 💡 What I Learned

* DHCP reduces manual configuration effort
* Lease time controls IP reuse
* Router can act as DHCP server
* DNS configuration is part of DHCP
