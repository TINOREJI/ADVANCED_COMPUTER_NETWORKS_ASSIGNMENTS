## 📘 About This Assignment

This assignment focuses on building and testing different network topologies using **Cisco Packet Tracer**.

Instead of just learning theory, the goal here was to **actually design networks, connect devices, assign IPs, and verify communication using ping**.

Each topology helped in understanding how real-world networks are structured and how devices communicate.

---

## 🎯 What This Covers

* Basic device connections (PC ↔ PC, PC ↔ Router)
* Networking using Hub and Switch
* Understanding different topologies:

  * Bus
  * Ring
  * Star
  * Tree
* Verifying connectivity using `ping`

---

## 🛠️ Tools & Requirements

* Cisco Packet Tracer
* Basic knowledge of IP addressing
* Understanding of network devices (PC, Router, Switch, Hub)

## 🔗 Connectivity Testing

To verify connections, the following command was used:

```bash
ping <destination-ip>
```

### ✔ Expected Output

* `Reply from ...` → Connection successful
* `Request timed out` → Configuration issue

---

## 🔍 Key Observations

* Devices must be in the **same subnet** for direct communication
* Proper **IP configuration** is critical
* Switches improve efficiency compared to hubs
* Routers are required for inter-network communication
* Network design directly impacts performance and reliability
