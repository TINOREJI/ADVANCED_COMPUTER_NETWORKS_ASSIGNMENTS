## 📘 About This Project

This assignment is a simple hands-on introduction to network analysis using **Wireshark**.

Instead of just reading theory, the idea here was to actually **see how data flows over a network**. By capturing live packets, we can observe how protocols like TCP, UDP, DNS, and ICMP work in real time.

---

## 🎯 What I Learned

* How to capture live network traffic
* How different protocols appear in real packets
* How DNS resolves domain names into IP addresses
* Difference between TCP (reliable) and UDP (fast)
* How to filter and analyze packets efficiently

---

### 5. Applying Filters

To make analysis easier, I used filters like:

| Filter | Purpose           |
| ------ | ----------------- |
| `ip`   | View IP packets   |
| `tcp`  | View TCP traffic  |
| `udp`  | View UDP traffic  |
| `dns`  | View DNS queries  |
| `icmp` | View ping packets |

### 6. Packet Analysis

Clicking on a packet showed detailed layers:

* Ethernet → MAC addresses
* IP → Source & destination IP
* TCP/UDP → Ports and communication details
* Payload → Actual transmitted data

## 💡 Key Takeaways

* DNS is always the first step when visiting a website
* TCP ensures reliable communication (used in browsing)
* UDP is faster but less reliable
* ICMP is mainly used for diagnostics (like ping)
